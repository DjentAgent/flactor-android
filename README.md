# FlacTor Android

`FlacTor` - Android-приложение для поиска музыкальных торрент-раздач, выбора нужных файлов внутри `.torrent` и надежной фоновой загрузки.

Проект объединяет удобный пользовательский сценарий и инженерную надежность:
- два режима поиска (через Spotify и ручной);
- управление файлами внутри раздачи до начала скачивания;
- устойчивая фоновая загрузка с очередью, паузой и возобновлением;
- сохранение в системную медиатеку или выбранную папку.

## Что решает приложение

- Быстро находит релевантные раздачи по артисту и треку.
- Позволяет скачать не всю раздачу, а только нужные файлы.
- Дает прозрачный контроль загрузок: статусы, прогресс, групповые действия.
- Стабильно работает при нестабильной сети и смене состояния приложения.

## Ключевые возможности

### 1. Два режима поиска
- `Spotify Public` - поиск треков через публичный каталог Spotify.
- `Manual Torrent` - ручной поиск по артисту, треку и качеству.
- Выбранный режим сохраняется в `DataStore`.

### 2. Поиск и подбор раздач
- Пагинация (`Paging 3`) и задержка ввода запроса.
- Фильтр качества: `Lossless / Lossy`.
- Поддержка CAPTCHA (`HTTP 428`) с повтором после ввода.
- История ручных запросов: сохранение, удаление, очистка.

### 3. Работа с файлами в раздаче
- Загрузка и кэширование `.torrent` по `topicId`.
- Поддержка раздач с одним файлом и с деревом папок.
- Экран `ManageFiles` включает поиск по файлам, фильтры по состояниям, режимы `List` и `Explorer`, массовый выбор и автоопределение наиболее подходящего трека.

### 4. Очередь и менеджер загрузок
- Локальное хранение задач в `Room`.
- Группировка по раздачам и датам.
- Состояния: `Active`, `Queued`, `Paused`, `Failed`, `Completed`.
- Действия: пауза, продолжение, отмена, повтор, удаление.
- Удаление записи с опцией удаления файла на устройстве.

### 5. Фоновая загрузка и устойчивость
- Работа через `Foreground Service`.
- Контроль параллельности (`Semaphore`), `WakeLock`.
- Повторные попытки, контроль зависаний, возврат в очередь при сбоях.
- Корректная финализация на 100% с проверкой фактического появления файла.

### 6. Сохранение файлов
- В `MediaStore` (медиатека) или в пользовательскую папку через `SAF`.
- Проверка уже существующих файлов перед записью.
- Поддержка постоянного доступа к выбранной папке (`persistable URI permission`).

### 7. Уведомления
- Канал `download_channel`.
- Сводное уведомление с количеством активных задач и скоростью.
- Быстрый переход в менеджер загрузок.
- Поддержка разрешения `POST_NOTIFICATIONS`.

## Архитектура

Проект построен на `Clean Architecture + MVVM`:
- `ui` - экраны Compose, `ViewModel`, `UiState`;
- `domain` - модели, контракты, сценарии использования;
- `data` - сеть, локальное хранилище, DataStore и подсистема загрузки;
- `di` и `navigation` - зависимости (`Hilt`) и маршрутизация.

Принципы разработки:
- `SOLID`;
- `KISS`;
- бизнес-логика вынесена из интерфейса в отдельные сценарии и сервисные компоненты.

## Основной поток загрузки

1. Пользователь выбирает раздачу и нужные файлы.
2. Задачи сохраняются в `Room` и попадают в очередь.
3. `DownloadService` планирует выполнение с учетом лимитов.
4. `DownloadTaskProcessor` ведет загрузку, обновляет прогресс, обрабатывает ошибки.
5. Готовый файл переносится в целевое хранилище (`MediaStore`/`SAF`), статус фиксируется как `COMPLETED`.

## Сборка и запуск

Требования:
- `JDK 17`
- `Android SDK 34`
- актуальная версия `Android Studio`

Команды:

```bash
./gradlew :app:compileDebugKotlin
./gradlew :app:testDebugUnitTest
./gradlew :app:detekt
./gradlew :app:ktlintCheck
```

Опциональные параметры в `gradle.properties`:
- `SPOTIFY_API_BASE_URL`
- `BACKEND_BASE_URL`
